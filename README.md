# Documentação UML

A UML (Unified Modeling Language) é uma linguagem visual usada para modelar, documentar e comunicar sistemas de software. Ela não é uma metodologia, mas sim um conjunto padronizado de diagramas que ajudam a representar desde a estrutura até o comportamento de um sistema. A UML facilita a vida de equipes de desenvolvimento porque:

📘 Documenta o sistema de forma clara e compreensível

🧠 Apoia o entendimento de requisitos e regras de negócio

🛠️ Ajuda no planejamento da arquitetura e do design

🤝 Melhora a comunicação entre desenvolvedores, analistas e stakeholders

🗂️ Principais tipos de diagramas

A UML é dividida em dois grandes grupos:

1. Diagramas Estruturais (mostram o que o sistema é):

Diagrama de Classes;
Diagrama de Objetos;
Diagrama de Componentes;
Diagrama de Pacotes;
Diagrama de Implantação.

3. Diagramas Comportamentais (mostram como o sistema se comporta):

- Diagrama de Casos de Uso;
- Diagrama de Sequência;
- Diagrama de Atividades;
- Diagrama de Estados;
- Diagrama de Comunicação.

Por que a UML continua relevante mesmo com metodologias ágeis e ferramentas modernas, a UML ainda é valiosa porque:

- É padrão internacional;
- É independente de linguagem e tecnologia;
- Ajuda a reduzir ambiguidades.

Funciona bem tanto em projetos grandes quanto pequenos.
